# Docs/Overview

## Structure
**Guides**
`docs/guides`: Guides for admin, dev and user. Use:
- `admin.md`, if you want to run chronicle.
    - contains: start/stop stack, config/env, DB access, troubleshooting
- `dev.md`, if you want to develop chronicle.
    - contains: local setup, tests, migrations, API, file structure
- `user.md`, if you want to use chronicle.
    - contains: core features, usage, smart scheduling, sync, FAQ

**Architecture**
`docs/architecture/decisions`: Decisions of the arcitecture in order to understand them.
